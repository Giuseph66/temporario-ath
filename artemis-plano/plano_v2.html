02 · Mapa de Variáveis de Ambiente

O que fica no ENV vs. vai para o banco

Regra: chaves por-tenant vão para o banco. Chaves de infraestrutura ficam no ENV.

Variável Destino Motivo
DATABASE_URL Mantém no ENV Infraestrutura — necessário para conectar ao banco antes de qualquer query
JWT_ACCESS_SECRET, JWT_REFRESH_SECRET Mantém no ENV Segurança do servidor — único por deployment
PORT, NODE_ENV Mantém no ENV Configuração de runtime do servidor
RESPONDI_WEBHOOK_SECRET Mantém no ENV Serviço global, não por-tenant
GEMINI_API_KEY → Banco (já migrado) Já existe como tenant.geminiApiKey — ENV é fallback
EVOLUTION_BASE_URL, EVOLUTION_API_KEY → Banco (já migrado) Já existem no tenant — Settings atual já gerencia
ASAAS_API_KEY, ASAAS_BASE_URL, ASAAS_WEBHOOK_SECRET → Banco (V2) Cada tenant tem conta Asaas própria — configurar via Integrações
META_ACCESS_TOKEN, META_PHONE_ID, META_VERIFY_TOKEN → Banco (V2) Cada tenant tem conta Meta Business própria
GOOGLE_CALENDAR_ID → Banco (V2) Cada tenant usa calendário próprio para agendamentos

04 · Mapa de Telas

Todas as telas da plataforma

6 rotas autenticadas + 3 rotas públicas. Todas precisam estar 100% funcionais no V2.

Dashboard

/dashboard
  • 4 cards: leads, matriculados, conversas, conversão
  • Status do agente + toggle ativo/inativo
  • Feed: últimas 5 atividades
  • API: GET /api/metrics + GET /api/agent

Conversas

/conversas
  • Lista lateral: avatar, nome, preview, timestamp
  • Busca por nome/telefone
  • Painel direito: timeline de msgs (user/assistant)
  • API: GET /api/conversations + /api/leads/:id

Leads

/leads
  • Tabela: nome, tel, status badge, última interação
  • Filtro por status + busca por nome/tel
  • Modal: detalhes + histórico + mudar estado
  • Exportar JSON (LGPD Art. 15)
  • API: GET /api/leads + PATCH state + DELETE

Integrações

/integracoes
  • Cards: Evolution, Meta, Asaas, Google Calendar
  • Status badge por integração (ativo/pendente)
  • Formulário inline por card
  • Evolution: QR code + status da instância
  • API: GET /api/integrations + PATCH por provider

Agente

/agente
  • Identidade: nome, role, número WhatsApp, modelo Gemini
  • Programas: lista add/edit/delete com preços
  • Configurações: settingsJson editor
  • Toggle ativo/inativo
  • API: GET+PATCH /api/agent/*

Configurações

/config
  • Conta: nome empresa, slug, plano
  • Gemini API Key (único campo de chave aqui)
  • Zona de perigo: exportar + sair
  • Nota: Evolution foi para /integracoes
